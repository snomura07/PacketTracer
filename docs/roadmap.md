# 開発ロードマップ

## 方針

最初からPacket Tracerを完全再現しない。

まずは、社内ネットワークの理解と障害切り分け練習に必要な、L3到達性判定に絞る。

## Phase 0: プロジェクト初期化

### 目的

Laravel + React + React Flowで開発を開始できる状態にする。

### 作業

- Laravelプロジェクト作成
- Inertia + React導入
- React Flow導入
- MySQL接続設定
- Docker開発環境の検討
- トップ画面表示

### 完了条件

ブラウザでアプリを開き、空のネットワークエディタ画面が表示される。

## Phase 1: トポロジー編集

### 目的

GUI上に機器を配置し、接続できるようにする。

### 作業

- PCノード追加
- Switchノード追加
- Routerノード追加
- Internetノード追加
- ノードのドラッグ移動
- ノード間の接続
- 構成の保存
- 構成の読み込み

### 完了条件

機器を配置し、線で接続し、再読み込み後も構成が復元される。

## Phase 2: IP設定

### 目的

各機器にIP設定を持たせる。

### 作業

- PCのIP、サブネットマスク、デフォルトゲートウェイ設定
- ルータの複数インターフェース設定
- ルータの静的ルート設定
- 設定パネル作成
- 入力バリデーション

### 完了条件

PCとルータにIP情報を設定し、DBに保存できる。

## Phase 3: pingシミュレーション

### 目的

端末間の通信可否を判定する。

### 作業

- IP計算処理
- 同一ネットワーク判定
- デフォルトゲートウェイ確認
- ルーティングテーブル探索
- 次ホップ探索
- 戻り経路確認
- シミュレーションAPI作成
- 結果表示パネル作成

### 完了条件

PC-AからPC-Bにpingを実行し、成功または失敗理由を表示できる。

## Phase 4: 障害切り分け練習

### 目的

設定ミスを含む演習シナリオを作れるようにする。

### 作業

- サンプルトポロジー作成
- 障害シナリオ作成
- 原因候補表示
- ヒント表示
- 正解確認

### 完了条件

デフォルトゲートウェイ未設定、サブネットマスク誤り、戻りルート不足などの演習ができる。

## Phase 5: 拡張機能

Phase 1〜4が安定してから検討する。

- VLAN
- NAT
- DHCP
- DNS
- ACL
- Link up/down
- 複数プロジェクト管理
- 構成図エクスポート

## 最初のマイルストーン

最初の実装目標は以下とする。

```text
PC-A ---- SW-1 ---- Router-1 ---- SW-2 ---- PC-B
```

この構成で、PC-AからPC-Bへpingできるかを判定する。

### 成功ケース

```text
PC-A: 192.168.10.10/24 GW 192.168.10.1
Router-1 eth0: 192.168.10.1/24
Router-1 eth1: 192.168.20.1/24
PC-B: 192.168.20.10/24 GW 192.168.20.1
```

### 失敗ケース

- PC-Aのデフォルトゲートウェイ未設定
- PC-Aのデフォルトゲートウェイが別ネットワーク
- Router-1のeth1未設定
- PC-Bのデフォルトゲートウェイ誤り
- 戻り経路不足
