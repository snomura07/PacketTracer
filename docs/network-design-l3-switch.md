# L3スイッチ設計方針

このドキュメントでは、PacketTracerにおけるL3スイッチの設計方針を定義する。

## 基本方針

- L2スイッチとL3スイッチは同じ `switch` Deviceを使用する。
- `metadata_json.switch_mode` により `l2` / `l3` を切り替える。
- 初期実装では物理ポートへIPを設定する現在の実装を許容する。
- ただし、将来的にはSVI（Switch Virtual Interface）へ移行できる構造を維持する。

## L2スイッチ

初期実装ではスイッチングハブと同一の扱いとする。

役割は以下のみ。

- 同一L2セグメントの転送
- MAC学習（将来実装）

VLAN・STP・LACP等は後続フェーズで実装する。

## L3スイッチ

L3スイッチはL2機能に加えルーティング機能を持つ。

Routerのように各物理ポートへIPを設定する考え方ではなく、将来的にはSVIへIPを持たせる。

例：

- VLAN10 → 192.168.10.1/24
- VLAN20 → 192.168.20.1/24

## デフォルトゲートウェイとデフォルトルート

PCはデフォルトゲートウェイを持つ。

例：

- PC-A: 192.168.10.10/24
- Default Gateway: 192.168.10.1

L3スイッチはルーティング機器であるため、デフォルトゲートウェイではなくデフォルトルートを持つ。

例：

- 0.0.0.0/0 → 192.168.254.1

## 実装上の注意

- L3スイッチをRouterそのものとして扱わない。
- 将来SVIへ移行できるよう責務を分離する。
- VLAN実装前はInterfaceを暫定利用してよい。
- 将来追加するVLAN・SVI・SwitchPortを見据えて設計する。
