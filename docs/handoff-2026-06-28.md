# Handoff Notes

## 現在の状態

このリポジトリは、Packet Tracer 風のネットワーク到達性シミュレータの MVP を実装中。

現時点では以下まで完了している。

- Laravel 11 + Inertia + React + TypeScript + React Flow の開発土台
- Docker Compose ベースの最小ローカル実行環境
- `NetworkProject` / `Device` / `NetworkCloud` / `DeviceInterface` / `Link` / `RouteEntry` の永続化モデル
- トポロジー保存 / 再読込 API
- Ping シミュレーション API の初版
- React Flow 上でのノード追加、削除、ドラッグ移動、接続
- ノード編集モーダル
- 右クリックメニューによる機器追加 / ノード削除
- Ping 結果表示
- UI の主要文言の日本語化
- 明るい配色テーマ
- `ProjectToolbar` / `PingPanel` / `TopologyContextMenu` などの分割開始

ローカル `main` は `origin/main` の PR #5 まで取り込んだ状態。

現在の作業ブランチは `feat/ping-panel-fixes` で、HEAD は `29e566e` (`Merge branch 'main' into feat/ping-panel-fixes`)。

## 現在の制約

- `PingSimulator` は next-hop を再帰的にたどる構成にはまだなっていない
- Master'sONE 判定も厳密な経路探索ではなく、MVP 寄りの簡略実装
- `NetworkEditor.tsx` は分割を始めたが、ノード編集モーダルやキャンバス周りの責務はまだ大きい
- Ping API 自体は保存済み DB の `id` を前提にしているため、未保存状態では実行できない
- `docker/node/Dockerfile` まわりは整理余地あり
