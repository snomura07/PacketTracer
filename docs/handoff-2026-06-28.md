## 実装上の注意点

### Link 制約

`links` テーブルでは DB の `check constraint` を使わず、`app/Models/Link.php` の `saving` フックで制約を掛けている。

理由:

- Laravel schema builder でそのまま portable に扱いづらかった
- SQLite / MySQL 差異をこの段階で避けたかった

したがって、今後 `Link` を保存するコードではこの制約を前提にすること。

### テスト DB

`phpunit.xml` は test 時に `sqlite` の `:memory:` を使う設定へ変更済み。

### .env

`.env` を追加済み。
現状はローカル SQLite 前提で、`DB_DATABASE=/app/database/database.sqlite` になっている。

Docker 経由で動かす場合はこの値を適宜見直すこと。
