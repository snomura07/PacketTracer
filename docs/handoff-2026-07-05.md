## 次のタスク

次は [docs/network-design-l3-switch.md](/home/snomura/workspace/PacketTracer/docs/network-design-l3-switch.md:1) を正として、L3 スイッチの実装をそれに合わせて修正すること。

重要な論点:

- `l2_switch` / `l3_switch` を別 `DeviceType` として持つ今の実装は、doc の方針とズレている
- doc では `switch` 1種 + `metadata_json.switch_mode` で `l2` / `l3` を切り替える
- L3 スイッチは Router と同一視せず、将来 `SVI` へ移行できる責務分離を維持する
- VLAN 実装前は暫定的に interface 利用を許容するが、最終的な設計は `SVI` 前提

次回の着手順推奨:

1. `docs/network-design-l3-switch.md` を正としてドメイン差分を整理する
2. `DeviceType` を `switch` へ寄せるか、互換レイヤで吸収するか決める
3. `NetworkProjectTopologyService` の normalize ロジックを設計方針に合わせて見直す
4. `PingSimulator` の `isSwitchport` / `isSvi` / `effectiveType()` 依存を点検する
5. UI 上の機器追加・表示・編集を `switch_mode` ベースへ揃える
6. 既存保存データ互換を壊さない移行方針を決める
